## 1. 현재 솔루션 기능 요약

### 1.1 티켓 관리
- 고객 웹 폼 기반 티켓 생성 (첨부파일, CAPTCHA, 우선순위 지원)
- 티켓 번호 형식: `TKT-{YYYY}-{NNNNNN}`
- 상태 관리: OPEN → IN_PROGRESS → WAITING → RESOLVED → CLOSED
- 우선순위: URGENT, HIGH, MEDIUM, LOW
- 상담원 수동/자동 배정 (카테고리 전문성 + 부하 분산 알고리즘)
- 티켓 이관 (사유 기록)
- 내부 메모 (고객 비공개)
- 댓글 스레드 (공개/내부)
- 활동 로그 (상태 변경, 배정, 이관 이력)
- 고급 검색 API (전문 검색, 다중 필터, 정렬, 페이지네이션)

### 1.2 사용자 관리
- **상담원**: CRUD, 역할(ADMIN/AGENT), 최대 티켓 수, 부재 관리, 비활성화 시 자동 재배정
- **고객**: 이메일 기반 식별, 티켓 이력 추적, AI 기반 분석
- **팀**: 팀 생성/관리, 리더 지정, 팀 기반 티켓 라우팅

### 1.3 커뮤니케이션
- 이메일 인바운드/아웃바운드 (SMTP, AWS SES, Resend)
- 이메일 스레딩 (Message-ID, In-Reply-To, References)
- 이메일 웹훅 (SendGrid, Mailgun, AWS SES, Postmark 형식)
- GitHub 웹훅 (커밋/PR/이슈 연동)

### 1.4 응답 템플릿
- CRUD, 카테고리/요청유형 연결
- 변수 치환 (`{{ticket.subject}}`, `{{customer.name}}` 등)
- 스마트 추천 (점수 기반 랭킹)
- 사용 이력 추적

### 1.5 SLA 관리
- 우선순위별 SLA 정책 (첫 응답/해결 시간)
- SLA 시계 (RUNNING/PAUSED/STOPPED)
- 업무시간 캘린더 (공휴일, 근무일 설정)
- 위반 감지 및 경고 (80% 임계값)

### 1.6 Git 연동
- GitHub/GitLab 지원
- 이슈 검색/생성/연결
- 커밋 메시지에서 티켓 번호 파싱
- 비동기 작업 큐 (재시도, 에러 추적)
- AES-256-GCM 토큰 암호화

### 1.7 보안
- 슬라이딩 윈도우 + 토큰 버킷 레이트 리밋
- 파일 업로드 검증 (MIME, 크기, 파일명 새니타이징)
- Zod 스키마 유효성 검사
- JWT 기반 티켓 접근 (서명 쿠키, 24시간 만료)
- CAPTCHA (Cloudflare Turnstile)
- 감사 로그 (액터, 액션, 리소스, 이전/이후 값, IP, UA)
- Helmet.js 보안 헤더

### 1.8 인증/인가
- NextAuth.js v5 (Credentials, Google, GitHub, SAML)
- RBAC (ADMIN, AGENT)
- 초기 비밀번호 변경 강제
- SAML SSO (BoxyHQ SAML Jackson)

### 1.9 기타
- 브랜딩 커스터마이징 (로고, 색상, 텍스트, 커스텀 CSS)
- LLM 설정 (Ollama, Google Gemini)
- 감사 로그 엑셀 내보내기
- 파일 저장 (로컬/AWS S3)
- 요청 유형 관리 (채널, 기본 우선순위, 기본 팀)

### 1.10 기술 스택
- Next.js 15 App Router + React 19 + TypeScript
- Prisma ORM 6.7.0 + SQLite 3
- Tailwind CSS + shadcn/ui (Radix UI)
- Server Actions + API Routes

## 2. 갭 분석 상세

### 2.1 옴니채널 지원 — 심각도: 높음

현재는 **웹 폼 + 이메일**만 지원. 업계 표준은 5개 이상 채널 통합.

| 부족 기능 | 설명 | 업계 사례 |
|-----------|------|----------|
| **실시간 라이브챗 위젯** | 웹사이트에 삽입하는 실시간 채팅 위젯. 고객이 페이지 이탈 없이 상담 가능 | Zendesk Chat, Intercom Messenger, Freshdesk Freshchat |
| **AI 챗봇** | 라이브챗 진입 전 자동 응답. 지식베이스 기반 FAQ 답변, 티켓 생성 안내 | Zendesk Answer Bot, Freshdesk Freddy |
| **카카오톡/LINE 연동** | 한국/일본 시장에서 필수. 카카오 비즈메시지 API 연동 | Zendesk Sunshine, 채널톡 |
| **전화(CTI) 연동** | VoIP/PSTN 통화 녹음, IVR, 콜백 기능 | Zendesk Talk, Freshdesk Contact Center |
| **SMS 알림** | 티켓 상태 변경 시 SMS 발송 | Twilio 연동 |
| **SNS 채널** | Twitter DM, Facebook Messenger 등 소셜 미디어 인바운드 처리 | Zendesk Social Messaging |
| **인앱 메시징 SDK** | 모바일 앱에 삽입하는 SDK | Intercom Mobile SDK, Zendesk SDK |

**구현 권장 순서**: 라이브챗 → AI 챗봇 → 카카오톡 → SMS → 나머지

---

### 2.2 지식베이스 / 셀프서비스 — 심각도: 높음

지식베이스가 없으면 모든 문의가 티켓으로 유입되어 상담원 부하가 증가함. 업계 데이터에 따르면 지식베이스 도입 시 **티켓 20~40% 감소** 효과.

| 부족 기능 | 설명 |
|-----------|------|
| **FAQ / 도움말 센터** | 카테고리별 정리된 문서 페이지. 공개/내부 문서 구분 |
| **문서 계층 구조** | 카테고리 > 섹션 > 문서 형태의 트리 구조 |
| **검색 기능 (고객용)** | 전문 검색. AI 기반 관련 문서 추천 |
| **티켓 생성 시 관련 문서 추천** | 고객이 제목/내용 입력 시 관련 지식베이스 문서를 실시간 제안 |
| **문서 유용성 평가** | "이 문서가 도움이 되었나요?" 피드백 수집 |
| **문서 조회수 추적** | 인기 문서, 검색 후 미해결 키워드 분석 |
| **SEO 최적화** | 공개 도움말 페이지의 메타태그, 사이트맵 자동 생성 |
| **커뮤니티 포럼** | 고객 간 질문/답변 게시판 |

**구현 권장 순서**: 문서 CRUD + 카테고리 → 검색 → 티켓 생성 시 추천 → 피드백/통계 → 포럼

---

### 2.3 AI / 자동화 — 심각도: 높음

현재 LLM 인프라(Ollama, Gemini)가 있지만 고객 분석에만 사용 중. 확장 여지가 큼.

| 부족 기능 | 설명 |
|-----------|------|
| **AI 답변 자동 제안** | 상담원이 티켓을 열면 AI가 답변 초안을 제안. 템플릿 + 지식베이스 + 이전 티켓 학습 |
| **AI 티켓 자동 분류** | 제목/내용 분석으로 카테고리, 우선순위, 담당 팀을 자동 설정 |
| **AI 감정 분석** | 고객 메시지의 감정 상태(긴급/불만/보통) 감지. 긴급 에스컬레이션 트리거 |
| **AI 티켓 요약** | 긴 스레드를 한 줄로 요약. 이관받은 상담원이 빠르게 파악 |
| **AI 번역** | 다국어 고객 응대 시 실시간 번역 |
| **자동화 규칙 엔진 (트리거)** | "조건 → 액션" 규칙. 예: 우선순위가 URGENT이고 30분 미응답 → 팀장에게 알림 |
| **시간 기반 자동화** | 예: 7일간 고객 무응답 → 자동 종료, 24시간 미배정 → 에스컬레이션 |
| **SLA 위반 자동 에스컬레이션** | SLA 임박/위반 시 상위 관리자에게 자동 알림 및 재배정 |
| **시각적 워크플로우 빌더** | 드래그앤드롭으로 자동화 규칙을 구성하는 UI |

**구현 권장 순서**: AI 답변 제안 → AI 분류 → 트리거 엔진 → 시간 기반 자동화 → 감정 분석 → 워크플로우 빌더

---

### 2.4 고객 경험 — 심각도: 중간

| 부족 기능 | 설명 |
|-----------|------|
| **고객 포털 (로그인 기반)** | 고객이 회원가입/로그인하여 자신의 모든 티켓을 한 곳에서 관리. 현재는 티켓번호+이메일 조회만 가능 |
| **고객 티켓 목록** | 로그인한 고객이 자신의 전체 티켓 이력을 조회 |
| **만족도 조사 (CSAT)** | 티켓 해결 후 1~5점 만족도 평가 + 코멘트. 이메일로 설문 링크 발송 |
| **NPS 설문** | 주기적 고객 만족도 측정 (0~10점) |
| **실시간 상태 알림** | 이메일 외에 브라우저 푸시/모바일 푸시로 티켓 상태 변경 알림 |
| **다국어 UI (i18n)** | 한국어 외 영어, 일본어 등 다국어 지원. next-intl 등 활용 |
| **모바일 반응형 최적화** | 현재 부분적. 모바일 사용자를 위한 완전한 반응형 |

**구현 권장 순서**: CSAT → 고객 포털/로그인 → 다국어 → 모바일 최적화

### 2.5 상담원 생산성 도구 — 심각도: 중간

| 부족 기능 | 설명 |
|-----------|------|
| **티켓 병합** | 동일 고객의 중복 티켓을 하나로 합치기. 댓글/첨부파일 통합 |
| **티켓 분할** | 하나의 티켓에 여러 이슈가 있을 때 분리 |
| **관련 티켓 연결** | 관련된 티켓을 링크로 연결. 부모-자식 관계 지원 |
| **충돌 감지** | 두 상담원이 동시에 같은 티켓을 편집할 때 경고 |
| **@멘션** | 댓글에서 다른 상담원을 @멘션하여 알림 |
| **팔로워/CC** | 티켓에 참관자를 추가. 진행상황을 자동으로 받아보기 |
| **시간 추적** | 티켓당 상담원이 소요한 시간 기록. 성과 측정 기반 |
| **사이드바 고객 정보** | 티켓 상세 화면 옆에 고객의 이전 티켓, 조직 정보, 주문 내역 등 표시 |
| **저장된 뷰/필터 UI** | DB 모델(SavedFilter)은 존재하나 UI 미구현. 개인/공유 뷰 |
| **키보드 단축키** | 빠른 상태 변경, 답변, 내부 메모 전환 등 |

**구현 권장 순서**: 티켓 병합 → 관련 티켓 연결 → @멘션 → 충돌 감지 → 저장된 뷰 UI → 시간 추적

### 2.6 리포팅 & 분석 — 심각도: 중간

현재는 기초 대시보드와 감사 로그 엑셀 내보내기만 존재.

| 부족 기능 | 설명 |
|-----------|------|
| **핵심 KPI 대시보드** | 첫 응답 시간(FRT), 평균 해결 시간(ART), SLA 준수율, CSAT 점수 |
| **상담원별 성과 리포트** | 처리 건수, 평균 응답/해결 시간, 만족도 점수 |
| **SLA 준수율 리포트** | 기간별 SLA 달성/위반 건수 및 비율 |
| **채널별 유입 분석** | 웹/이메일/API 등 채널별 티켓 유입량 추이 |
| **시간대별 분석** | 피크 타임 식별, 상담원 스케줄 최적화 근거 |
| **커스텀 리포트 빌더** | 사용자가 차원/지표를 선택하여 커스텀 보고서 생성 |
| **예약 리포트** | 매주/매월 자동으로 리포트를 이메일로 발송 |
| **실시간 대시보드** | WebSocket 기반 실시간 갱신. 현재 대기 건수, 온라인 상담원 수 등 |

**구현 권장 순서**: 핵심 KPI → 상담원 성과 → SLA 준수율 → 채널 분석 → 커스텀 리포트

### 2.7 알림 & 실시간 — 심각도: 중간

| 부족 기능 | 설명 |
|-----------|------|
| **인앱 실시간 알림** | WebSocket 또는 SSE 기반. 관리 화면에서 새 티켓, 새 댓글 등 실시간 알림 벨 |
| **브라우저 푸시 알림** | Web Push API. 브라우저 탭이 닫혀있어도 알림 수신 |
| **Slack/Teams 연동 알림** | 특정 채널에 티켓 이벤트 알림. 슬랙에서 바로 상태 변경 가능 |
| **개인별 알림 설정** | 상담원이 수신할 알림 유형을 선택 (내 티켓만, 팀 티켓, 모든 긴급 등) |
| **실시간 티켓 목록 갱신** | 새로고침 없이 목록이 자동 업데이트 |

**구현 권장 순서**: 인앱 알림(SSE) → Slack 연동 → 브라우저 푸시 → 개인별 설정

### 2.8 통합 & 확장성 — 심각도: 중간

| 부족 기능 | 설명 |
|-----------|------|
| **API 문서 (OpenAPI/Swagger)** | REST API의 자동 생성 문서. 외부 연동 개발자를 위한 필수 사항 |
| **Webhook 발신** | 티켓 이벤트 발생 시 외부 URL로 HTTP POST. 커스텀 연동의 기반 |
| **Slack 연동** | 슬랙에서 티켓 생성/응답/상태 변경. Slack App |
| **Jira 연동** | Jira 이슈와 티켓 양방향 동기화 |
| **CRM 연동** | Salesforce, HubSpot 등 고객 정보 동기화 |
| **Zapier/n8n 연동** | 노코드 자동화 플랫폼 연동. Webhook 발신이 전제 |

**구현 권장 순서**: Webhook 발신 → API 문서 → Slack 연동 → Jira 연동

### 2.9 관리 & 운영 — 심각도: 낮음~중간

| 부족 기능 | 설명 |
|-----------|------|
| **커스텀 역할** | ADMIN/AGENT 외에 SUPERVISOR, VIEWER 등 세분화된 권한 |
| **멀티 브랜드** | 하나의 시스템에서 여러 브랜드의 헬프데스크 운영 |
| **데이터 가져오기/내보내기** | 타 시스템에서 마이그레이션. CSV/JSON 형태 |
| **커스텀 필드 UI** | DB 모델(CustomFieldDefinition/Value)은 존재하나 UI 미구현 |
| **IP 화이트리스트** | 관리자 접근 IP 제한 |

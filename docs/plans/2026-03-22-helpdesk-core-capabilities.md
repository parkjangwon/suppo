# Helpdesk Core Capabilities Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use test-driven-development while expanding each capability.

**Goal:** 현재 헬프데스크를 운영형 솔루션으로 끌어올리기 위해 SLA, 자동화, 운영 큐, 고객 360, 티켓 관계, 권한, 연동, 다국어, AI 자동화를 구체적으로 정의하고 제품 코드에 반영한다.

**Architecture:** 이미 존재하는 Prisma 모델과 API 조각을 재사용해 관리 콘솔에서 직접 운영할 수 있는 허브를 만든다. 신규 기능은 `운영 정책` 허브, 티켓 목록 프리셋 큐, 고객 360 타임라인, 티켓 관계 패널처럼 현재 사용자 경험에 바로 연결되는 형태로 제공한다.

**Tech Stack:** Next.js App Router, Prisma, shadcn/ui, Vitest, Playwright

---

## 기능 정의

### 1. SLA/응답 기한 관리

- 우선순위별 `첫 응답 시간`, `해결 목표 시간`을 정책으로 저장한다.
- 활성 정책은 우선순위별 1개만 유지한다.
- 티켓 목록에서 `SLA 임박`, `SLA 위반` 큐로 빠르게 진입한다.
- 티켓 상세와 대시보드에서 SLA 상태를 소비한다.

### 2. 자동화 규칙

- 트리거: `TICKET_CREATED`, `TICKET_UPDATED`, `TICKET_ASSIGNED`
- 조건: 상태, 우선순위, 이메일 패턴, 키워드, 담당자, 팀
- 동작: 상태 변경, 우선순위 변경, 담당자 재배정, 팀 재지정, 태그 추가, 이메일 알림
- 규칙은 우선순위 숫자로 실행 순서를 제어한다.

### 3. 저장된 보기 / 운영 큐

- 프리셋 큐: `오늘 처리`, `긴급`, `VIP`, `내 담당`, `SLA 임박`
- 개인 저장 보기와 공유 저장 보기를 함께 사용한다.
- 티켓 목록 URL 파라미터만으로 큐 상태를 재현할 수 있어야 한다.

### 4. 고객 360 타임라인

- 고객 프로필, 관리자 메모, AI 분석, 최근 티켓을 한 흐름으로 보여준다.
- 고객 상세 페이지는 상담 품질 향상 목적의 `운영 맥락`을 제공해야 한다.

### 5. 티켓 병합 / 분할 / 부모-자식

- 현재 단계에서는 `병합`과 `병합 이력`을 제품화한다.
- 다음 단계로 `분할`, `부모-자식 관계`를 동일한 패널에서 확장 가능하게 유지한다.

### 6. 세분화된 권한 관리

- 현재 운영 역할은 `관리자`, `상담원` 두 가지다.
- 이번 단계에서는 관리자 전용 화면을 명확한 안내 상태로 보호한다.
- 다음 단계에서 `팀장`, `읽기전용`을 추가할 수 있도록 운영 기준을 문서화한다.

### 7. 외부 연동/API/Webhook

- Git, 이메일, SAML, 웹훅을 `운영 정책` 허브에서 한눈에 파악한다.
- CRM, ERP, 모니터링, 배포 도구는 웹훅/API 확장 대상으로 정의한다.

### 8. 다국어

- 지원 언어는 `한국어(ko)`, `영어(en)` 두 가지만 운영한다.
- 이번 단계는 지원 범위와 운영 기준을 정의하고, 전역 i18n 확장 토대를 마련한다.

### 9. AI 자동 분류와 초안 생성

- 현재 제공 기능: 응답 초안, 티켓 요약, 지식 연결
- 확장 목표: 자동 태깅, 자동 요약, 자동 라우팅
- 운영 원칙: 완전 자동 응답보다 `상담원 보조`를 우선한다.

## 이번 구현 범위

- `운영 정책` 허브 추가
- SLA 정책 CRUD UI/API
- 자동화 규칙 CRUD UI/API
- 티켓 목록 프리셋 큐
- 고객 360 타임라인 카드
- 티켓 관계 패널
- 관리자 전용 기능 정의와 연동/다국어/AI 운영 기준 정리

## 다음 단계

- 팀장/읽기전용 역할을 위한 `AgentRole` 확장과 세부 권한 테이블 도입
- 다국어 문자열 사전 및 locale persistence 도입
- AI 자동 태깅/자동 라우팅을 티켓 생성 플로우에 연결
- 티켓 분할 및 부모-자식 관계 모델 추가
